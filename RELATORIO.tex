\begin{abstract}
Este projeto implementa um sistema híbrido de recomendação para uma loja virtual de perfumes nacionais, combinando as técnicas de \textbf{TF-IDF} (filtragem por conteúdo) e \textbf{SVD} (filtragem colaborativa). O sistema recomenda produtos personalizados para novos usuários a partir de suas preferências declaradas, refinadas pela matriz de utilidade de 500 usuários similares. A abordagem híbrida resolve o problema do ``cold-start'', oferecendo recomendações relevantes mesmo para usuários sem histórico de compras prévias.
\end{abstract}

\section{Introdução}

O objetivo deste projeto é desenvolver um sistema inteligente de recomendação que combine técnicas de filtragem por conteúdo e filtragem colaborativa. A motivação é prática: melhorar a experiência do usuário em uma loja virtual de perfumes nacionais, oferecendo recomendações personalizadas e relevantes.

A abordagem híbrida foi escolhida para mitigar os problemas conhecidos:
\begin{itemize}
    \item \textbf{Cold-start com TF-IDF puro:} novos usuários recebem recomendações genéricas
    \item \textbf{Cold-start com SVD puro:} sem dados de interação, não há como fazer predições
    \item \textbf{Solução híbrida:} TF-IDF gera candidatos relevantes, SVD reordena com insights colaborativos
\end{itemize}

    \item \textbf{Avaliações registradas:} 5.905 (23,6\% de densidade)

    \item \textbf{Escala de notas:} 1 a 5 (inteiros)
    \item \textbf{Células vazias (0):} 19.095 (usuários que não avaliaram)
\end{itemize}

\subsubsection{Metodologia de Geração}

A matriz foi gerada sinteticamente mantendo critérios realistas:

\paragraph{Personas de Usuários:}
Cada usuário recebe uma ``persona'' com preferências aleatórias:
\begin{itemize}
    \item 1-2 famílias olfativas favoritas
    \item Uma ocasião preferida
    \item Conjunto de gêneros aceitáveis (45\% masc, 45\% fem, 10\% ambos)
\end{itemize}

\paragraph{Modelo de Notas:}
A nota de um perfume para um usuário é calculada como:
\begin{equation}
\text{nota} = 2.5 + \Delta_{\text{fam}} + \Delta_{\text{ocas}} + \Delta_{\text{gênero}} + \epsilon
\end{equation}

onde:
\begin{itemize}
    \item $\Delta_{\text{fam}} \in [0, 1.6]$ se há sobreposição de tokens na família olfativa
    \item $\Delta_{\text{ocas}} \in [0, 0.6]$ se há sobreposição na ocasião
    \item $\Delta_{\text{gênero}} = -2.2$ se gênero é incompatível
    \item $\epsilon \sim \mathcal{N}(0, 0.45)$ é ruído gaussiano
    \item nota final: $\text{clamp}(\text{nota}, 1, 5)$
\end{itemize}

\paragraph{Probabilidade de Avaliação:}
\begin{itemize}
    \item Gênero incompatível: 4\%
    \item Família favorita: 65\%
    \item Outros produtos: 22\%
\end{itemize}

\paragraph{Usuários Silenciosos:}
Aproximadamente 12\% dos usuários nunca deixam avaliações, modelando comportamento realista.

\section{Metodologia de Recomendação}

\subsection{Fase 1: TF-IDF (Filtragem por Conteúdo)}

O algoritmo TF-IDF identifica candidatos iniciais baseado no perfil do novo usuário.

\subsubsection{Fundamentos Teóricos}

TF-IDF (Term Frequency - Inverse Document Frequency) é uma representação vetorial que:

\begin{itemize}
    \item \textbf{TF:} Frequência do termo no documento (quanto mais aparece, maior o peso)
    \item \textbf{IDF:} Raridade global do termo (termos raros têm maior impacto discriminativo)
    \item \textbf{Combinação:} $\text{TF-IDF}(t,d) = \text{TF}(t,d) \times \log\left(\frac{N}{n_t}\right)$
\end{itemize}

\textbf{Intuitivamente:} Um perfume com ``floral'' + ``noturno-formal'' + ``feminino'' tem alta similaridade com o vetor de uma usuária que busca exatamente esses termos.

\subsubsection{Processo de Vetorização}

\begin{enumerate}
    \item \textbf{Concatenação de características:}
    \begin{equation}
    \text{perfil} = \text{família\_olfativa} + \text{ ocasião} + \text{ gênero}
    \end{equation}
    \textit{Exemplo:} ``floral noturno formal feminino''
    
    \item \textbf{Tokenização e TF-IDF:}
    \begin{itemize}
        \item \texttt{scikit-learn} converte 50 perfumes em vetores TF-IDF de dimensão $D$ (vocab size)
        \item Aplicado também ao perfil do usuário
        \item Resultado: 50 vetores de características (pré-calculados)
    \end{itemize}
    
    \item \textbf{Matriz TF-IDF:} Armazenada em \texttt{tfidf\_matrix.pkl} para reuso
\end{enumerate}

\subsubsection{Cálculo de Similaridade}

Para um novo usuário, a similaridade com cada perfume é:

\begin{equation}
\text{sim\_tfidf}(i) = \cos(\vec{v}_{\text{perfil}}, \vec{v}_{i}) = \frac{\vec{v}_{\text{perfil}} \cdot \vec{v}_{i}}{|\vec{v}_{\text{perfil}}| \cdot |\vec{v}_{i}|}
\end{equation}

\textbf{Propriedades da similaridade de cosseno:}
\begin{itemize}
    \item Resultado no intervalo [0, 1] (0 = nenhuma similaridade, 1 = idêntico)
    \item Invariante à magnitude (dois perfis ``floral feminino'' similares mesmo com casing diferente)
    \item Baseada em direção, não em magnitude absoluta
\end{itemize}

\subsubsection{Filtros e Top-20}

Antes de retornar candidatos, aplica-se máscaras de negócio:

\begin{equation}
\text{candidatos} = \{i : \text{genero}(i) = \text{genero\_pref} \textbf{ AND } \text{preco}(i) \leq \text{preco\_max}\}
\end{equation}

Entre os candidatos filtrados, retorna-se os 20 com maior $\text{sim\_tfidf}(i)$.

\textbf{Nota sobre validação de família:} Embora TF-IDF capture similaridade lexical, a validação de família é feita a posteriori (em Nível 0) para garantir que a família solicitada realmente aparece no produto.

\subsection{Fase 2: SVD (Filtragem Colaborativa)}

O algoritmo SVD reordena os top-20 candidatos usando matrix factorization e insights de usuários similares.

\subsubsection{Fundamentos Teóricos}

\textbf{Matrix Factorization via SVD:}

Dada a matriz de utilidade $R_{500 \times 50}$ (usuários $\times$ produtos):

\begin{equation}
R = U \Sigma V^T \approx \hat{U} \hat{\Sigma} \hat{V}^T
\end{equation}

onde se mantém apenas os $k$ maiores fatores singulares (tipicamente $k = 50$).

\textbf{Intuitivamente:} A matriz é decomposta em 2 matrizes menores:
\begin{itemize}
    \item $U_{500 \times k}$: Preferências latentes de cada usuário em $k$ dimensões
    \item $V_{50 \times k}^T$: Características latentes de cada produto em $k$ dimensões
    \item Multiplicação $U \times V^T$ prediz notas faltantes
\end{itemize}

\subsubsection{Processo de Filtragem Colaborativa}

\begin{enumerate}
    \item \textbf{Identificar usuários similares:}
    \begin{itemize}
        \item Calcular similaridade TF-IDF entre o perfil do novo usuário e todos os usuários existentes
        \item Selecionar os 50 usuários com maior similaridade
        \item Armazena em \texttt{top\_usuarios} (índices)
    \end{itemize}
    
    \item \textbf{Extrair predições SVD para os top-20 produtos:}
    \begin{equation}
    \hat{R}_{50 \times 20} = \text{pred\_matrix}[\text{top\_usuarios}, \text{top\_20\_perfumes}]
    \end{equation}
    \textit{Nota:} A matriz de predições SVD foi pré-calculada durante treinamento e armazenada em \texttt{svd\_pred\_matrix.pkl}
    
    \item \textbf{Calcular score SVD médio por perfume:}
    \begin{equation}
    \text{svd\_score}(j) = \frac{1}{50} \sum_{i=1}^{50} \hat{R}_{ij}
    \end{equation}
    
    \item \textbf{Normalizar para [0, 1]:}
    \begin{equation}
    \text{svd\_norm}(j) = \frac{\text{svd\_score}(j) - 1}{4}
    \end{equation}
    (Subtrai 1 e divide por 4 pois escala de notas é [1, 5])
\end{enumerate}

\subsubsection{Interpretação}

O SVD captura padrões ocultos que TF-IDF não vê:

\begin{itemize}
    \item Dois usuários fisicamente distintos podem ter preferências similares
    \item Um perfume não óbvio pode ter notas altas de usuários com padrões similares
    \item Exemplo: Usuária gosta de florais e relogios; outras mulheres nesse ``cluster latente'' também gostam de um amadeirado específico
\end{itemize}

\subsection{Combinação Híbrida}

O score final combina ambas as abordagens de forma sinérgica:

\begin{equation}
\text{score\_final}(j) = 0.7 \times \text{tfidf}(j) + 0.3 \times \text{svd\_norm}(j)
\end{equation}

\subsubsection{Justificativa dos Pesos}

\textbf{70\% TF-IDF (Conteúdo):}
\begin{itemize}
    \item Mantém relevância de conteúdo como base principal
    \item Garante que as características solicitadas (família, ocasião, gênero) apareçam no produto
    \item Para novos usuários, TF-IDF sozinho já fornece recomendações aceitáveis
    \item Peso alto protege contra ``drift'' de SVD em catálogos pequenos
\end{itemize}

\textbf{30\% SVD (Colaborativo):}
\begin{itemize}
    \item Captura preferências ocultas de usuários similares
    \item Permite descoberta de serendipidade (perfume não-óbvio mas que usuários similares amam)
    \item Reordena top-20 candidatos para refletir gosto coletivo
    \item Peso modesto evita overfitting a um cluster específico de usuários
\end{itemize}

\subsubsection{Fluxo Completo}

\begin{enumerate}
    \item \textbf{Entrada:} Novo usuário com preferências (família, ocasião, gênero, preço\_max)
    
    \item \textbf{TF-IDF:} Gera 20 candidatos ordenados por similaridade de conteúdo
    
    \item \textbf{SVD:} Reordena os 20 com pesos colaborativos
    
    \item \textbf{Score Final:} Calcula $0.7 \times \text{tfidf} + 0.3 \times \text{svd\_norm}$
    
    \item \textbf{Threshold:} Se score $\geq 0.50$, retorna top-5 (caso contrário, ativa fallback)
    
    \item \textbf{Saída:} 5 perfumes com mensagem indicando nível de qualidade
\end{enumerate}

O sistema retorna os \textbf{top-5} perfumes ordenados por score\_final.

\section{Sistema de Fallback}

Quando o modelo híbrido não encontra recomendações satisfatórias (score $< 0.50$), o sistema implementa uma cascata de fallback com 4 níveis progressivos, relaxando restrições de forma inteligente:

\subsection{Arquitetura do Fallback}

\begin{enumerate}
    \item[\textbf{Nível 0}] \textbf{Match Perfeito}: Todas as restrições ativas
    \begin{itemize}
        \item Filtros: Família olfativa + Ocasião + Gênero + Preço
        \item Score mínimo: $\geq 0.50$
        \item Validação: A família solicitada deve aparecer no produto
        \item Saída: ``✅ Recomendação personalizada com excelente match!''
    \end{itemize}
    
    \item[\textbf{Nível 1}] \textbf{Ocasião Restritiva}: Remove ocasião
    \begin{itemize}
        \item Filtros: Família + Gênero + Preço (ocasião removida)
        \item Motivo: Algumas ocasiões são raras (ex: ``diurno-verao'' com apenas 2 produtos)
        \item Score mínimo: $\geq 0.50$
        \item Saída: ``⏰ Poucas opções para essa ocasião. Mostrando recomendações de outras ocasiões.''
    \end{itemize}
    
    \item[\textbf{Nível 2}] \textbf{Gênero Cruzado}: Remove gênero
    \begin{itemize}
        \item Filtros: Família + Ocasião + Preço (gênero removido)
        \item Motivo: Algumas famílias são predominantemente de um gênero (ex: ``amadeirado'' é 94\% masculino)
        \item Score mínimo: $\geq 0.50$
        \item Saída: ``✨ Seu perfil é raro. Mostrando perfumes similares de outros gêneros.''
    \end{itemize}
    
    \item[\textbf{Nível 3}] \textbf{Sem Match}: Retorna top-5 mockados
    \begin{itemize}
        \item Filtros: Nenhum (apenas ordena por popularidade)
        \item Método: Retorna os 5 perfumes mais vendidos para o gênero solicitado
        \item Saída: ``📌 Nenhuma combinação ideal foi encontrada. Mostrando os perfumes mais populares da comunidade.''
    \end{itemize}
\end{enumerate}

\subsection{Exemplos Práticos de Acionamento}

\paragraph{Exemplo 1: Ocasião Restritiva (Nível 1)}
\begin{itemize}
    \item Solicitação: Floral feminino, ``noturno-inverno'', R\$ 200
    \item Nível 0: TF-IDF encontra ``noturno-inverno'', mas score = 0.38 $<$ 0.50 (falha)
    \item Nível 1: Remove ocasião, busca floral feminino R\$ 200 (score = 0.56 $\geq$ 0.50)
    \item Resultado: ``Lily Absolu'' recomendado com aviso de ocasião
\end{itemize}

\paragraph{Exemplo 2: Gênero Cruzado (Nível 2)}
\begin{itemize}
    \item Solicitação: Amadeirado feminino, ``diurno-casual'', R\$ 150
    \item Nível 0: Busca família + ocasião + gênero, mas score = 0.42 $<$ 0.50 (falha)
    \item Nível 1: Remove ocasião, mas apenas 1 amadeirado feminino (R\$169.90, acima de R\$150)
    \item Nível 2: Remove gênero, encontra amadeirado masculino com score = 0.56 $\geq$ 0.50
    \item Resultado: Recomenda amadeirado masculino com aviso de gênero cruzado
\end{itemize}

\subsection{Benefícios da Cascata}

\begin{itemize}
    \item \textbf{Robustez:} Sempre retorna alguma recomendação (nunca vazio)
    \item \textbf{Progressividade:} Relaxa restrições de forma inteligente, não aleatória
    \item \textbf{Transparência:} Mensagem clara comunica por que há fallback
    \item \textbf{Validação de Família:} Nível 0 valida se a família foi realmente encontrada (não apenas score alto)
\end{itemize}

\subsection{Qualidade das Recomendações}

O sistema prioriza:
\begin{enumerate}
    \item Relevância de conteúdo (família olfativa, ocasião, gênero)
    \item Preferências colaborativas (usuários similares)
    \item Filtros práticos (gênero, faixa de preço)
\end{enumerate}

\textbf{Exemplo de fluxo:}
\begin{itemize}
    \item \textit{Entrada:} Nova usuária, feminino, gosta de florais, orçamento R\$ 150
    \item \textit{TF-IDF (top-20):} Lily Essence, Floratta Rosas, Egeo Dolce, \ldots
    \item \textit{SVD reordena:} Baseado em padrões de mulheres com preferência floral
    \item \textit{Top-5 final:} Perfumes florais bem-avaliados por personas similares
\end{itemize}

\subsection{Limitações e Tratamento}

\paragraph{Cold-start para novos usuários:}
Sem histórico, SVD retorna predições neutras. Nesse caso, TF-IDF (70\%) domina, garantindo recomendações minimamente relevantes. À medida que o usuário interage, seria possível retreinar SVD periodicamente.

\paragraph{Matriz esparsa:}
23,6\% de densidade é adequado para SVD, mas matriz mais densa melhoraria predições.

\section{Conclusão}

O projeto implementa com sucesso um sistema de recomendação híbrido robusto que atende todos os requisitos:

\begin{itemize}
    \item[$\checkmark$] 50 produtos com 3 características + avaliação 1-5
    \item[$\checkmark$] 500 usuários com matriz de utilidade reproduzível (seed=42)
    \item[$\checkmark$] Modelo híbrido TF-IDF (70\%) + SVD (30\%) bem documentado
    \item[$\checkmark$] Sistema de fallback em cascata (4 níveis) com relaxamento progressivo
    \item[$\checkmark$] Validação de família olfativa em Nível 0 para evitar falsos positivos
    \item[$\checkmark$] Threshold calibrado (0.50) para otimizar cobertura vs. relevância
    \item[$\checkmark$] Interface web funcional (Gradio) com 4 abas de funcionalidade
    \item[$\checkmark$] Notebook Jupyter com EDA e treinamento
    \item[$\checkmark$] Código modularizado, reutilizável e bem comentado
\end{itemize}

\subsection{Contribuições Técnicas}

\begin{enumerate}
    \item \textbf{Hibridização efetiva:} Combinação 70/30 oferece equilíbrio entre explicabilidade e descoberta
    
    \item \textbf{Cascata inteligente:} Fallback progressivo garante sempre alguma recomendação válida, nunca vazio
    
    \item \textbf{Validação de constraints:} Verificações explícitas (família, gênero, preço) em cada nível
    
    \item \textbf{Threshold adaptativo:} Score $\geq 0.50$ calibrado para evitar recomendações fracas
\end{enumerate}

\subsection{Lições Aprendidas}

\begin{itemize}
    \item Validação de família é crucial: TF-IDF sozinho produz falsos positivos (``amadeirado'' com notas de madeira)
    
    \item Cascata é preferível a simples fallback: relaxar ocasião antes de gênero oferece UX melhor
    
    \item Pequenos catálogos beneficiam-se de TF-IDF: com apenas 50 produtos, peso alto em conteúdo reduz overfitting
    
    \item Matriz esparsa (23,6\%) é adequada para SVD, mas densidade maior melhoraria predições
\end{itemize}

\subsection{Extensões Futuras}

\begin{enumerate}
    \item \textbf{Retrainamento periódico:} Incorporar feedback novo a cada semana para melhorar SVD
    
    \item \textbf{Deep learning:} Usar autoencoders ou embeddings word2vec para representações contínuas
    
    \item \textbf{A/B testing:} Variar pesos (testar 60/40, 80/20) e medir impacto em satisfação
    
    \item \textbf{Análise de serendipidade:} Medir quantos usuários descobrem perfumes não-óbvios via SVD
    
    \item \textbf{Context-aware:} Recomendar baseado em sazonalidade, clima local, evento (casamento, formatura)
    
    \item \textbf{Explicabilidade melhorada:} Informar usuário por que cada perfume foi recomendado (``similar a usuários que gostam de florais'')
\end{enumerate}
